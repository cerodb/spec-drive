## Codebase Patterns

### Shell Portability
- resolve-config.sh is the shared library sourced by all hook scripts (stop-watcher.sh and context-loader.sh source it with `. "$SCRIPT_DIR/resolve-config.sh"`)
- portable_realpath() priority: python3 → realpath → cd+pwd -P for dirs → dirname+cd+pwd -P for files
- Portable mtime check pattern: python3 os.path.getmtime (first) → stat -c %Y (Linux) → stat -f %m (macOS) → skip gracefully
- Avoid GNU-only find flags: -mmin, -printf, -exec {} \+ (not portable to macOS BSD find)
- All hooks use set -euo pipefail — prefer `|| true` guards on fallible non-critical paths
- Hooks bail early with `exit 0` (not `exit 1`) on non-fatal conditions to avoid breaking Claude sessions

### Testing
- test-hooks.sh uses bash `source .` to test functions from resolve-config.sh inline
- mktemp -d for tmpdir creation; EXIT trap for cleanup
- Ambiguity safety tests create multiple state files and verify hook output text

### Architecture
- Hook scripts are bash-only, no compilation
- resolve-config.sh: config resolution utilities + portable helpers
- stop-watcher.sh: Stop hook (drives execution loop continuation)
- context-loader.sh: SessionStart hook (outputs active project context to stderr)
